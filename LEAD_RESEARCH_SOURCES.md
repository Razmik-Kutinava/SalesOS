# Источники для поиска лидов и компаний — черновик на апрув

**Статус:** список для согласования с владельцем продукта и юристом.  
**Правило SalesOS:** публичный скрейпинг только с **allowlist доменов** и с учётом ToS; платные API — по договору и ключам.

---

## 1. Обогащение контактов и компаний (B2B, API / платформы)

| Категория | Примеры | Заметка |
|-----------|---------|---------|
| Контакты + компании | [Apollo](https://www.apollo.io/), [ZoomInfo](https://www.zoominfo.com/), [Lusha](https://www.lusha.com/), [Clearbit](https://clearbit.com/) | Обычно подписка/API; не «парсинг сайта», а лицензированные данные. |
| Email-файндинг | [Hunter.io](https://hunter.io/), [Snov.io](https://snov.io/) | Уже в контуре PRD (`docs/INTEGRATIONS.md` — Snov/Hunter). |
| Технологии стека | [BuiltWith](https://builtwith.com/), [Wappalyzer](https://www.wappalyzer.com/) (API/планы) | Для приоритизации ICP. |

**Апрув:** какие вендоры допустимы по бюджету и ПДн.

---

## 2. Поиск и «сигналы» (не браузер)

| Категория | Примеры | Заметка |
|-----------|---------|---------|
| Веб-поиск API | [Serper](https://serper.dev/), [Tavily](https://tavily.com/), Brave Search API | Для запросов «компания + отрасль», новости. |
| Новости / триггеры | Google Alerts (ручной), RSS отраслевых СМИ | Осторожно с автоматизацией Google. |

**Апрув:** один основной поисковый API (в стеке уже задуман Serper).

---

## 3. Открытые справочники и реестры (варианты по региону)

| Регион / тип | Примеры | Заметка |
|----------------|---------|---------|
| РФ | ЕГРЮЛ/ФНС (открытые сведения), СПАРК-интерес (платно) | Не все данные можно кэшировать без проверки. |
| Международные | OpenCorporates, SEC EDGAR (для публичных компаний США) | Часто есть API или выгрузки. |
| Отзывы / репутация | G2, Capterra (публичные страницы продуктов) | ToS — читать; лучше официальный экспорт/API. |

**Апрув:** какие юрисдикции и типы записей нужны в MVP.

---

## 4. Соцсети и профессиональные сети

| Источник | Заметка |
|----------|---------|
| LinkedIn | PRD явно исключает обход ToS с логином; официальные продукты Sales Navigator / API — отдельный бюджет и правила. |
| Telegram / X | Только публичные каналы/посты и при соблюдении политики платформы. |

**Апрув:** обход «в лоб» без официального API — **не рекомендуется**.

---

## 5. Где уместен Playwright (отдельный воркер)

- Статические/SSR страницы **разрешённых** доменов из allowlist (например, сайт компании, страница вакансий, публичный прайс).
- Не заменяет API обогащения; используется, когда нужен **рендер JS** или нет стабильного API.

**Апрув:** финальный список **hostname** для `PLAYWRIGHT_FETCH_ALLOWED_HOSTS` (и отдельно — ваш список сайтов, который вы дадите позже).

---

## 6. Рекомендованный порядок внедрения

1. **Поиск API** (Serper и т.д.) + **импорт CSV** (уже в приложении).  
2. **Один** вендор обогащения по API (после апрува).  
3. **Playwright** только по allowlist + аудит в БД (джоба `FetchUrlJob` + воркер в репозитории).
