# Telegram Bot API

## 1. Назначение

**Telegram Bot API** — HTTP-интерфейс для создания **ботов** в мессенджере Telegram: приём и отправка сообщений, медиа, кнопки, inline-режим, платежи (в ограниченных сценариях), управление вебхуками. Бот выступает как **автоматизированный аккаунт**, управляемый вашим сервером.

## 2. Создание бота

Через **@BotFather** создаётся токен `123456:ABC...`. Токен — секрет; хранится в переменных окружения или секрет-хранилище. Утечка токена даёт полный контроль над ботом.

## 3. Режимы получения обновлений

- **Long polling**: `getUpdates` в цикле с таймаутом; просто без публичного HTTPS.
- **Webhook**: Telegram POST-ит JSON на ваш URL; нужен валидный TLS-сертификат (или ngrok для разработки).

## 4. Формат обновления (Update)

Содержит одно из: `message`, `edited_message`, `callback_query` (нажатие inline-кнопки), `inline_query`, и др. Для голосовых сообщений в `message` присутствует `voice` с `file_id`.

## 5. Отправка сообщений

`sendMessage`, `sendPhoto`, `sendDocument` и т.д. Параметры: `chat_id`, текст, `parse_mode` (HTML/MarkdownV2), `reply_markup` для клавиатур.

## 6. Клавиатуры

**ReplyKeyboardMarkup** — кнопки под полем ввода. **InlineKeyboardMarkup** — кнопки под конкретным сообщением с `callback_data` или URL.

## 7. Медиа и файлы

`getFile` + `https://api.telegram.org/file/bot<token>/<file_path>` для загрузки бинарника (голос, документ). Ограничения размера файлов по типу сообщения.

## 8. Голосовые сообщения

`voice` — OGG Opus; длительность и `file_id` для последующей загрузки. Сервер скачивает файл и передаёт в ASR (Whisper и др.).

## 9. Команды

`/start`, пользовательские `/help` — объявляются через BotFather (`setMyCommands`) для UX. Парсинг текста `/command@botname args`.

## 10. Лимиты и флуд-контроль

Слишком частые сообщения в один чат могут получить **429 Too Many Requests**; нужны backoff и очередь. Глобальные лимиты на методы описаны в документации.

## 11. Безопасность

- Проверка, что webhook запросы от Telegram (секретный токен в пути или заголовке, IP allowlist при возможности).
- Не логировать полные токены.
- Валидация `chat_id` для админских команд.

## 12. Приватность

Сообщения проходят через инфраструктуру Telegram; для строгой приватности учитывать политику мессенджера. Бот видит только диалоги, где пользователь написал боту или добавил в группу.

## 13. Группы и супергруппы

`chat_id` отрицательный для групп. Права бота, анти-спам, упоминания — отдельная логика.

## 14. Inline-режим

Пользователь @бот в любом чате — запросы на ваш сервер для подстановки результатов.

## 15. Deep linking

`https://t.me/your_bot?start=payload` передаёт параметр в `/start` — для онбординга и трекинга источника.

## 16. Интеграция с бэкендом

Rails/Python/Node поднимают контроллер `/telegram/webhook`, парсят JSON, ставят фоновые джобы на тяжёлую обработку, отвечают `200 OK` быстро (Telegram ретраит при ошибках).

## 17. Конкурентность

Одинаковые `update_id` при ретраях; идемпотентная обработка или дедупликация по id.

## 18. Локализация

Поле `language_code` у пользователя — подсказка для мультиязычных ответов.

## 19. Тестирование

Локально: long polling или tunnel; стейджинг с реальным webhook.

## 20. Заключение

Bot API — стандартный способ мобильного **пульта** и уведомлений. Для голоса критичны загрузка файла, ASR и безопасная интерпретация команд.

## Дополнение: логирование PII

Usernames и тексты сообщений — персональные данные; минимизировать логи в продакшене.

## Дополнение: Bot API vs MTProto

Bot API — HTTP к серверам Telegram; MTProto — низкоуровневый клиентский протокол, не смешивать задачи.

## Дополнение: FSM для диалогов

Конечный автомат состояний в Redis/БД для многошаговых сценариев без потери контекста.

## Дополнение: throttle пользователя

Защита от флуда команд с per-user rate limit.

## Дополнение: inline keyboards UX

Короткий `callback_data` (лимит длины) — хранить id сущности, не весь payload.

## Дополнение: пересылка в support-чат

Группа админов с ботом для эскалации сложных запросов.

## Дополнение: BotFather настройки

Описание команд, картинка бота, privacy mode в группах — влияет на видимость сообщений.

## Дополнение: анимации и стикеры

Отдельные методы API для файлов большого размера; не смешивать с текстовой логикой команд.

## Дополнение: супергруппы и topic forums

Идентификаторы тредов в форумах — расширенные поля `message_thread_id`.

## Дополнение: миграция чата в supergroup

Смена `chat_id` при апгрейде — учитывать в сохранённых привязках пользователей.
