# Serper.dev и поисковые API

## 1. Назначение

**Serper.dev** — коммерческий **Google Search API**: программный доступ к результатам поисковой выдачи в структурированном JSON без прямого скрейпинга HTML Google. Используется для мониторинга упоминаний, OSINT, обогащения сущностей, агрегации новостей в автоматизированных пайплайнах.

## 2. Модель использования

Регистрация, получение **API-ключа**, HTTP POST с телом JSON (запрос, параметры локали, страницы, тип поиска). Ответ содержит **organic results**, иногда **answer box**, **related searches**, **knowledge graph** в зависимости от провайдера и запроса.

## 3. Квоты и биллинг

Тарификация обычно за **количество запросов**; есть бесплатный trial/лимитированный план. Превышение — 402/429 или аналог — нужны retry с backoff и кэширование.

## 4. Легальность и ToS

Использование только через официальный API провайдера; соблюдать **условия Serper** и политики Google относительно автоматизированного доступа. Результаты не для перепродажи сырой выдачи в некоторых сценариях — читать договор.

## 5. Интеграция в приложение

Серверное приложение вызывает API из **фоновых джобов** с секретом в env; не вызывать с клиента браузера (утечка ключа). Таймауты HTTP, идемпотентность джобов.

## 6. Кэширование

Повторяющиеся запросы кэшировать по хешу параметров с TTL — экономия денег и rate limits.

## 7. Качество выдачи

Персонализация и геолокация влияют на результаты; параметры `gl`, `hl` задают страну и язык интерфейса.

## 8. Альтернативы

**SerpAPI**, **Scale SERP**, **Bing Web Search API**, **Brave Search API** — разные цены, покрытие и лимиты.

## 9. Ограничения

Не гарантируется полнота индекса как у ручного браузера; некоторые блоки (реклама) отсутствуют в API.

## 10. Обработка результатов

Парсинг URL, title, snippet; дедупликация; фильтрация по доменам; глубина обхода связанных страниц — логика приложения.

## 11. Наблюдаемость

Логировать latency, коды ответов, стоимость в условных единицах для финансового контроля.

## 12. Тестирование

Стубы HTTP в unit-тестах; контрактные тесты на staging с реальным API осторожно.

## 13. Безопасность

Ротация ключей, минимальные права, мониторинг аномального объёма запросов (утечка ключа).

## 14. Этика

Не использовать для спама, манипуляции ранжированием или нарушения приватности физлиц.

## 15. Заключение

Serper — удобный слой к программному поиску Google; критичны квоты, кэш и соблюдение условий использования.
