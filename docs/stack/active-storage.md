# Active Storage: файлы в Rails

## 1. Назначение

**Active Storage** — подсистема Rails для загрузки файлов в **облачные** или **локальные** хранилища с унифицированным API. Поддерживает **прямую загрузку** с клиента в облако (minimize нагрузки на app server), **варианты изображений** (thumbnails), **анализ** содержимого (опционально).

## 2. Основные сущности

**Blob** — метаданные файла (ключ, checksum, content type, размер). **Attachment** — связь между моделью и blob (`has_one_attached`, `has_many_attached`).

## 3. Сервисы хранения

Адаптеры: **Disk** (локальный диск), **S3** (и совместимые MinIO, Wasabi), **Azure**, **GCS**. Конфигурация в `config/storage.yml` и `config/environments/production.rb`.

## 4. Прямая загрузка

Клиент запрашивает **signed URL**, загружает файл напрямую в S3, затем подтверждает в Rails — экономия RAM и таймаутов на больших файлах.

## 5. Варианты (variants)

Для изображений: `resize_to_limit`, `resize_to_fill` через **image_processing** gem и **libvips** или ImageMagick — генерируются по требованию или заранее по политике.

## 6. Предпросмотры PDF/видео

**preview** для генерации кадра/страницы — зависит от внешних утилит на сервере.

## 7. CDN

Публичные URL blob через proxy или прямой CDN origin для разгрузки приложения.

## 8. Безопасность

Не отдавать приватные вложения публично без **авторизации**; использовать `rails_blob_path` с контроллером-прокси или signed expiring URLs.

## 9. Миграции

Таблицы `active_storage_blobs`, `active_storage_attachments`, `active_storage_variant_records` создаются установочной миграцией.

## 10. Удаление

`purge` удаляет файл из хранилища и записи БД; фоновые джобы для очистки orphaned blobs при сбоях.

## 11. Валидации

Размер и content-type на уровне модели через кастомные валидаторы или гемы.

## 12. Тестирование

Fixture файлы в `test`/`spec`; сервис **test** с хранением в tmp.

## 13. Производительность

Кэширование variant; фоновая обработка тяжёлых преобразований Active Job.

## 14. Сравнение с CarrierWave/Shrine

Active Storage — дефолт Rails без отдельной колонки на каждое поле файла в классическом стиле; гибкость Shrine выше в некоторых продвинутых сценариях.

## 15. Заключение

Active Storage закрывает типовые задачи вложений; для публичных PDF в RAG важны checksum, идемпотентная индексация и права доступа.
