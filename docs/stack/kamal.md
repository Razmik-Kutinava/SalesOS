# Kamal: деплой контейнеров без Kubernetes

## 1. Назначение

**Kamal** (ранее MRSK) — инструмент от 37signals для **деплоя Docker-контейнеров** на один или несколько серверов по SSH с **zero-downtime** переключением, **прокси** Traefik и управлением **секретами**. Позиционируется как альтернатива Kubernetes для небольших и средних команд.

## 2. Компоненты

**Конфигурация** в `config/deploy.yml`: образ registry, сервисы, хосты, env, volumes, аксессоры. Команды CLI `kamal setup`, `kamal deploy`, `kamal app logs`. На сервере поднимается **Traefik** для маршрутизации и TLS (Let's Encrypt).

## 3. Registry

Образы пушатся в **Docker Hub**, GHCR, ECR и др.; Kamal логинится и пуллит на хостах.

## 4. Роли серверов

**Primary** и **replicas** для горизонтального масштаба; **accessories** для сопутствующих контейнеров (БД, Redis) при желании co-locate.

## 5. Секреты

`kamal env push` или интеграция с **1Password**, **Dotenv** на этапе деплоя; секреты не в git.

## 6. Миграции

Хуки `pre-deploy` / отдельные команды для `rails db:migrate` в одноразовом контейнере перед переключением трафика.

## 7. Zero-downtime

Новый контейнер поднимается, healthcheck проходит, Traefik переключает вес на новый инстанс, старый останавливается.

## 8. Откат

Деплой предыдущего тега образа; хранение нескольких версий для быстрого rollback.

## 9. Требования

SSH-доступ, Docker на хостах, открытые порты 80/443 для ACME; DNS указывает на IP сервера.

## 10. Ограничения

Не оркестратор уровня K8s: нет встроенного autoscaling по метрикам, сложные сетевые политики — вручную.

## 11. Безопасность

SSH-ключи, firewall, обновление хостов; сканирование образов до пуша.

## 12. Мониторинг

Внешние агенты (Uptime, Prometheus node exporter); Kamal даёт логи и статус контейнеров.

## 13. Сравнение с Capistrano

Capistrano — деплой без Docker; Kamal предполагает **immutable образы**.

## 14. Сравнение с Docker Swarm/K8s

Меньше абстракций и YAML-манифестов; выше для простых топологий.

## 15. Лицензия

MIT — открытый исходный код.

## 16. Заключение

Kamal — практичный путь доставки контейнеризованных Rails-приложений на VPS/облачные VM без полного стека Kubernetes.

## Приложение: Thruster

В экосистеме Rails 8 Thruster может обслуживать статику и сжатие перед приложением; совместимость с Kamal через сервис-описание в конфиге.

---

## Приложение: несколько регионов

Kamal не даёт глобального load balancing из коробки — DNS/geo routing внешними средствами.

---

## Приложение: база данных

Обычно управляемый RDS/Cloud SQL **вне** Kamal; accessory только для малых стеков.

---

## Приложение: CI

Pipeline собирает образ, пушит тег, вызывает `kamal deploy` с runner секретами.

---

## Приложение: стоимость

Экономия на control plane K8s при цене собственной эксплуатации VM.

## Дополнение: IPv6
