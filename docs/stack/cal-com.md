# Cal.com: планирование встреч

## 1. Назначение

**Cal.com** (ранее Calendly-like open-source продукт) — платформа **самобронирования** встреч: пользователь выбирает свободный слот из календаря владельца, получает подтверждение, создаётся событие в **Google Calendar**, **Microsoft Outlook** или другом провайдере при интеграции. Есть облачный SaaS и **self-hosted** вариант.

## 2. Типы событий

**Event types** задают длительность (15/30/60 мин), буферы между встречами, минимальное время до брони, вопросы гостю, локацию (ссылка Zoom/Meet), количество участников.

## 3. Доступность

Синхронизация с календарём владельца для **исключения** занятых слотов; учёт часовых поясов гостя и хоста.

## 4. API

REST API для программного создания бронирований, управления типами событий, webhooks на подтверждение/отмену. Аутентификация API-ключом или OAuth в зависимости от развёртывания.

## 5. Встраивание

**Embed** виджет на сайт или ссылка в email — стандартный сценарий «запишитесь на звонок».

## 6. Webhooks

Уведомления приложению о `BOOKING_CREATED`, `BOOKING_CANCELLED` — для обновления CRM и напоминаний.

## 7. Брендинг

Кастомизация страниц, домен, white-label на платных планах self-hosted/enterprise.

## 8. Командный режим

Round-robin между менеджерами, коллективная доступность.

## 9. Безопасность

Защита от спама бронированиями (капча, ограничения); не публиковать API-ключи в клиенте.

## 10. Конфиденциальность

Персональные данные гостей (email, имя) — обработка по GDPR при EU-аудитории.

## 11. Сравнение с Calendly

Cal.com отличается открытым исходным кодом и возможностью self-host; функциональность пересекается.

## 12. Self-hosted

Требует Node.js, PostgreSQL, фоновые джобы; обновления и бэкапы на стороне оператора.

## 13. Интеграции

Видеоконференции, платежи (опционально), Zapier/Make — в зависимости от экосистемы.

## 14. Лимиты

Облачные планы ограничивают число календарей/пользователей; API rate limits.

## 15. Заключение

Cal.com — стандартный способ снизить трение «письмо → встреча» через персональную ссылку и автоматическое создание события.
