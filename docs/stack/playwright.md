# Playwright: автоматизация браузера

## 1. Назначение

**Playwright** — библиотека от Microsoft для **End-to-End тестирования** и **автоматизации** Chromium, Firefox и WebKit единым API. Управляет реальным браузером: навигация, клики, ввод текста, ожидания, сеть, файлы, мобильную эмуляцию.

## 2. Архитектура

Драйвер запускает **браузерный процесс** и общается по протоколу (CDP для Chromium). API доступен для **Node.js**, **Python**, **Java**, **.NET**. Тесты изолированы **контекстами** (аналог инкогнито-профилей) с отдельными cookie/storage.

## 3. Селекторы

Рекомендуемая стратегия: **role**, **text**, **test-id** (`data-testid`) — устойчивее, чем хрупкие CSS/XPath от автогенерации вёрстки. `locator` API с **авто-ожиданием** видимости и действия.

## 4. Ожидания

Playwright **автоматически ждёт** перед действиями (actionability): элемент виден, стабилен, не перекрыт. Явные `waitFor` для кастомных условий. Таймауты настраиваются глобально и локально.

## 5. Сеть

**route** / **intercept** запросов для моков API, блокировки ресурсов, подмены ответов — ускорение тестов и изоляция от внешних сервисов.

## 6. Скриншоты и видео

Артефакты при падении тестов; **trace viewer** для пошаговой отладки (скрин, DOM, сеть).

## 7. Параллельный запуск

Workers в конфиге; шардирование в CI для ускорения пайплайна.

## 8. Генерация кода

**Codegen** записывает действия пользователя в скрипт — стартовая точка для тестов (требует ручной чистки селекторов).

## 9. CI

Официальные Docker-образы с браузерами и зависимостями; кэширование браузерных бинарников (`npx playwright install`).

## 10. Не только тесты

**Скрейпинг** и RPA: headless-режим, persistent context с логином, обход простых антиботов — на свой риск и с соблюдением ToS сайтов.

## 11. Headless vs headed

Headless быстрее в CI; headed нужен для отладки и некоторых сайтов с детекцией headless.

## 12. Мобильная эмуляция

Профили устройств, viewport, touch, geolocation — не заменяют тест на реальном устройстве, но покрывают типовые баги вёрстки.

## 13. Безопасность

Автологин с сохранением state в файле — защищать артефакты CI. Секреты не хардкодить в репозиторий.

## 14. Стабильность тестов

Flaky-тесты из-за анимаций, гонок, внешних CDN — фиксируют `clock`, стабилизируют данные, избегают sleep в пользу ожиданий по условию.

## 15. Сравнение с Selenium

Playwright чаще быстрее и с лучшими ожиданиями из коробки; Selenium — зрелая экосистема и Grid.

## 16. Сравнение с Cypress

Cypress — другой архитектурный модель (один браузер в ране); Playwright — мультибраузер и параллельность.

## 17. Версионирование

Привязка версии `@playwright/test` к версии браузеров; обновлять синхронно.

## 18. Лицензия

Apache 2.0 — дружелюбна к коммерческому использованию.

## 19. Ограничения

Не лезет в **shadow DOM** без специальных приёмов; **iframe** требуют переключения контекста; некоторые сайты блокируют автоматизацию.

## 20. Заключение

Playwright — современный стандарт E2E; понимание **locators**, **ожиданий** и **изоляции сети** критично для поддерживаемых тестов и надёжной автоматизации.
