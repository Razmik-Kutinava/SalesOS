# Участие в разработке SalesOS

Краткие правила для коммитов, веток и качества кода.

## Ветки

- `main` — стабильная линия.
- Фичи: `feature/kratkoe-opisanie` или `feat/...`.
- Исправления: `fix/...`.

## Коммиты

- Сообщения на **русском или английском** — единообразно в рамках репозитория.
- Формат по желанию: префиксы `feat:`, `fix:`, `docs:`, `chore:`.
- Один логический коммит — одна цель.

## Pull request

- Описание: **что** сделано и **зачем**.
- Самопроверка: тесты (`bin/rails test`), линтер, быстрый ручной сценарий при изменении UX.

## Ruby / Rails

- Следовать существующему стилю файлов.
- Запуск тестов: `bin/rails test` (при необходимости `bin/ci`).

## Линтер

- RuboCop: `bin/rubocop` или `bundle exec rubocop`.

## Секреты

- Не коммитить `.env`, ключи API, дампы БД с реальными ПДн.
- Примеры конфигурации — только с плейсхолдерами.

Спасибо за аккуратные изменения.
